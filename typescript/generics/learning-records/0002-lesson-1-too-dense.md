# Lesson 1 was too dense to follow

User explicitly said the first lesson was not easy enough to understand. The lesson introduced too many ideas at once (type parameters, inference, arrow function gotcha, multiple code examples) before the core concept had landed.

**Implications:** Future lessons must lead with a single real-world analogy before any syntax. Limit to one new concept per lesson. Cut code examples to the minimum needed to demonstrate that one concept. Quiz only on what was explicitly taught in that lesson.
