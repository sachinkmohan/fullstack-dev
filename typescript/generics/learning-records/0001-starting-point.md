# Starting point established

User knows JavaScript well but is near-beginner on TypeScript. Their goal is interview readiness for TypeScript generics specifically — not deep library authorship. They can use `Array<T>` and `Promise<T>` but haven't written their own generic functions.

**Implications:** Start from first principles (what problem do generics solve?) before any syntax. Avoid assuming knowledge of TS basics like interfaces or type aliases — introduce them only as needed to explain generics. Every lesson should feel directly connected to "this will come up in an interview."
