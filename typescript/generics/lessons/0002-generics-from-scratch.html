Generics from scratch

~8 minutes  ·  One concept only: what is a type parameter?

This lesson has one job: make <T> feel obvious. Nothing else.

Start with something you already know

You've written this in JavaScript before:

function giveMeBack(value) {
  return value;
}

It works with anything:

giveMeBack("hello")  // returns "hello"
giveMeBack(42)       // returns 42
giveMeBack(true)     // returns true

Now TypeScript asks: what type does this function return?

The honest answer is: it depends on what you pass in.

The problem with guessing

You could type it as string:

function giveMeBack(value: string): string {
  return value;
}

But now it only works with strings. Pass a number and TypeScript gives an error.

You could use any:

function giveMeBack(value: any): any {
  return value;
}

This compiles — but you've told TypeScript "stop checking, I don't care." The return type is any, so TypeScript can't help you after this point. In an interview, this answer will get pushed back on.

The problem: you want to say "whatever type goes in, that same type comes out" — without picking a specific type in advance.

The solution: a placeholder for the type

Generics let you use a placeholder. You write <T> — where T just means "some type, to be decided later":

function giveMeBack<T>(value: T): T {
  return value;
}

Read this out loud: "This function takes some type T. It receives a value of that type. It returns a value of that type."

Now when you call it, TypeScript fills in T automatically:

you call with
"hello"
T becomes
string
you call with
42
T becomes
number
you call with
true
T becomes
boolean
const a = giveMeBack("hello");  // TypeScript knows: a is string
const b = giveMeBack(42);        // TypeScript knows: b is number
const c = giveMeBack(true);      // TypeScript knows: c is boolean

You never wrote string, number, or boolean in the function. TypeScript figured it out from the argument.

T is just a name — like a variable

Just like you name a variable x or count, you name a type parameter T. The name doesn't matter — T is just the convention everyone uses (short for "Type").

// These are all identical:
function giveMeBack<T>(value: T): T { return value; }
function giveMeBack<Banana>(value: Banana): Banana { return value; }
function giveMeBack<Whatever>(value: Whatever): Whatever { return value; }

In interviews, always use T. It signals you know the convention.

You've been using generics all along

Look at these — you've written them before:

const names: Array<string> = ["Alice", "Bob"];
const ids:   Array<number> = [1, 2, 3];

Array<string> means: "an Array where T = string". The <string> is you explicitly telling TypeScript what the placeholder is. The only new thing in this lesson is that you can now write your own functions that use the same pattern.
