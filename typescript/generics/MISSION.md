# Mission: TypeScript Generics

## Why
To pass technical interviews that test TypeScript knowledge. When asked to write or explain generic functions, utility types, or constrained type parameters on the spot, be able to do so with confidence and correct syntax.

## Success looks like
- Write a generic function from scratch without syntax errors
- Explain what `T`, `extends`, and `keyof` do when asked by an interviewer
- Read a piece of generic library code and describe what it does
- Implement common interview patterns: `pick`, `identity`, `readonly wrapper`, `ReturnType`
- Spot the difference between `T extends object` and `T extends {}` and explain it

## Constraints
- Starting from beginner TS — knows JavaScript well
- Goal is interview readiness, not deep library authorship
- Lessons must be short and completable quickly

## Out of scope
- Advanced distributive conditional types
- Template literal types
- Building a full typed SDK or library
- `infer` keyword (until fundamentals are solid)
