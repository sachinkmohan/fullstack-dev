The Practice Stack — for full-stack

Quick reference · what to solve, where, and why · stop practicing randomly

Two rules:  (1) Practice by topic/pattern, never randomly.  (2) Match the rung to the skill you're leveling — katas alone won't make you full-stack.
RungTrainsWhereUse when
1 · Language fluency Idiomatic JS/TS, small algorithms, syntax reflexes Exercism (mentored), Codewars (focused sets) Warm-ups; new language/feature; rusty syntax
2 · DS & algorithms Pattern recognition: two pointers, sliding window, hash maps, recursion LeetCode Patterns, NeetCode 150 Go by pattern, ~3–5 problems per pattern, not random
3 · Frontend build Turning real designs into working, responsive UI Frontend Mentor Sharpening the "F" half of full-stack
4 · Real apps (biggest lever) Full-stack integration: API, DB, auth, deploy, debugging real systems build-your-own-x, your own shipped projects Most of an intermediate dev's time
5 · Read & contribute Reading unfamiliar code, conventions, architecture Open source; source of libraries you use Deposits senior-level chunks; long arc

Stage map (find roughly where you are)

  • Foundations → rung 1 dominant. Goal: fluency.
  • Algorithms → rung 2 by pattern. Goal: recognize, not memorize.
  • Application → rungs 3–4. Goal: ship real things end-to-end. (~2.5-yr intermediate dev usually lives here.)
  • Mastery → rungs 4–5 + system design. Goal: handle ambiguity & scale.

Make any rung deliberate

  • Target one skill at the edge of your ability.
  • Stretch with a problem/project that forces it.
  • Feedback fast: tests, mentor, error, code review.
  • Review: write the pattern you learned (bank the chunk).
