Stop practicing randomly: the practice stack

~12 minutes  ·  For an intermediate dev (≈2.5 yrs) aiming at full-stack

You're doing the right activity — practicing — but in the way that wastes the most effort: randomly. Random Codewars katas mostly retest fluency you already have, and rarely land on the edge of your ability, which is the only place deliberate practice works (Lesson 2).

The evidence: learners who practice by pattern rather than randomly report dramatically higher success — one analysis cites roughly 85% vs 35%. (DesignGurus) Random reps feel productive; patterned reps actually build the library.

The second mistake: katas ≠ full-stack

Codewars and LeetCode train one muscle: language fluency and algorithmic patterns. That matters — but a full-stack developer also needs to build UIs, wire up APIs and databases, handle auth, deploy, and debug real systems. No amount of katas teaches those. You have to match the practice type to the skill you're leveling.

Think of it as a stack of rungs. Each trains a different layer of "full-stack."

RUNG 1 · LANGUAGE FLUENCY

Idiomatic JS/TS & small algorithms

Syntax reflexes and clean, idiomatic code. This is what Codewars trains — fine as a warm-up, but you've largely got this after 2.5 years.

Where: Exercism (free human mentor review of your approach) · Codewars in focused sets.

RUNG 2 · DATA STRUCTURES & ALGORITHMS

Patterns, not random problems

Two pointers, sliding window, hash-map lookups, recursion. Do 3–5 problems per pattern back-to-back so the pattern becomes a chunk — don't shuffle.

Where: LeetCode Patterns (Sean Prashad) · NeetCode 150.

RUNG 3 · FRONTEND BUILD

Real designs → working UI

Turn professional design briefs into responsive, accessible interfaces. Sharpens the "F" half of full-stack with realistic constraints.

Where: Frontend Mentor.

RUNG 4 · REAL APPS — YOUR BIGGEST LEVER

End-to-end integration

API + database + auth + deploy + debugging a system you built. This is where "intermediate" becomes "full-stack." Ship small, real projects; rebuild real software to understand it.

Where: build-your-own-x · your own shipped side projects.

RUNG 5 · READ & CONTRIBUTE

Other people's code

Read the source of a library you use; fix a small open-source issue. Deposits senior-level chunks — architecture, conventions, trade-offs. This is the long arc (Norvig).

Where: open source · the source of your daily dependencies.

You are (probably) here → At ~2.5 years aiming full-stack, your highest-leverage time is Rung 4 (real apps) plus Rung 2 by pattern. Treat Rung 1 as warm-ups, not the main event. Use roadmap.sh/full-stack to spot which specific Rung-4 skills (auth, caching, testing, CI/CD…) you're missing.

The one habit that makes any rung count: apply the deliberate loop — pick a target at your edge, get fast feedback, and review the pattern you learned. Random difficulty on any rung still wastes reps.

Your turn: build a 4-week plan

Replace "random Codewars" with this. Write it down:

  1. One Rung-4 project you'll ship in 4 weeks (small & real — e.g. a notes app with auth + a database).
  2. One Rung-2 pattern per week (e.g. week 1: sliding window → 4 problems from the patterns list).
  3. 10-min warm-up on Rung 1 before each session (optional).
  4. Weekly review: what got you stuck? That gap is next week's target.
