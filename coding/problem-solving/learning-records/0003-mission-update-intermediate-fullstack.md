2. **End goal made explicit:** become a **full-stack web/app developer**. The problem-solving workspace is now framed as serving that career goal.

## The trigger
"What resources should I focus on solving... I've been solving on Codewars *randomly*. What should help me level up at various stages?"

## Key insight
The learner is doing the right activity (practice) the wrong way: **randomly**. This contradicts the deliberate-practice principle from Lesson 2 — random katas rarely hit the edge of ability, and mostly retrain language fluency the learner already has. Topic/pattern-based practice strongly outperforms random grinding (one cited figure: ~85% vs ~35%).

Second insight: for a **full-stack** goal, katas (Codewars/LeetCode) train only a slice (language fluency + algorithmic chunks). Leveling up requires matching *practice type* to *skill*: fluency → algorithms → real projects → reading/contributing.

## Decision
Lesson 3 = "The practice stack": a staged map of *what to solve and where*, replacing random Codewars with deliberate, goal-matched practice. Anchored to the deliberate-practice loop (Lesson 2) and the four-step loop (Lesson 1).

## ZPD note
As an intermediate dev, the highest leverage is likely **real full-stack projects + targeted DS&A by pattern**, not more easy katas. Lesson positions the learner there.
