# Mission: Problem Solving (thinking like a coder)

## Why
To stop freezing when a coding problem feels new. The goal isn't to memorize Fibonacci or any single puzzle — it's to own a **repeatable thinking process** that turns "I have no idea where to start" into a calm, mechanical sequence of steps. Solving problems faster is a side effect of solving them *deliberately*.

The trigger: a Fibonacci problem that "seemed not functional." The real lesson hiding inside it is *how to approach any problem* — when to use pen/paper vs. the editor, when to write pseudocode, and how to experiment toward a solution instead of guessing.

## Success looks like
- When given a new problem, run a fixed 4-step loop instead of jumping straight to typing code
- Restate any problem in your own words and write 3+ concrete input→output examples *by hand* before coding
- Write a plan in plain-language pseudocode before touching syntax
- Translate pseudocode into working JavaScript/TypeScript
- Catch your own bugs by testing code against the hand-made examples (the "look back" step)
- Know *deliberately* when to reach for pen/paper, plain text, or the editor — and why

## End goal (career)
Become a strong **full-stack web/app developer**. This problem-solving workspace serves that goal: sharper problem-solving + the right practice resources are the engine that moves an intermediate dev toward full-stack mastery.

## Constraints
- Level: **Intermediate — ~2.5 years professional web-dev experience** (updated 2026-06-17; earlier "beginner+" was incorrect)
- Language: JavaScript / TypeScript (matches this monorepo)
- Currently practices on Codewars *randomly* — wants a deliberate, staged practice plan instead
- Goal is durable competence toward full-stack, not interview cramming or competitive programming
- Lessons must be short and completable quickly

## Out of scope (for now)
- Big-O / time & space complexity analysis (will come once the loop is a habit)
- Specific algorithm families (sorting, graphs, dynamic programming theory)
- Data-structure deep dives
- Competitive-programming speed tricks
