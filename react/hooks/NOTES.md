# Teaching Notes

## Lesson pacing
- One concept per lesson. Resist the urge to introduce a second.
- Lead with an analogy or a JavaScript fact the user already knows, before any React-specific syntax.
- Minimum code. Cut every example that doesn't directly demonstrate the one concept.
- Quiz only on what was explicitly taught in this lesson — never on adjacent ideas.

## Style
- Same visual template as the TypeScript generics workspace (Georgia serif body, mono code, blue accent).
- Each lesson links a single primary source. Read it after the lesson, not before.
- Each lesson ends with: "Ask your teacher in the chat" — encourages followup questions.

## Interview framing
- This is interview prep, so every lesson should surface the "why" — the underlying mental model — not just the API.
- Where relevant, name the classic interview question this concept unlocks.
