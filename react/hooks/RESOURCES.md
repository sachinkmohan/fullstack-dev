# React Hooks Resources

## Knowledge

- [React Docs — "Reacting to Input with State" through "Escape Hatches"](https://react.dev/learn)
  The official, rewritten 2023 docs. Tone is excellent, mental models are correct. Use for: anything — this is the primary source.
- [React Docs — "You Might Not Need an Effect"](https://react.dev/learn/you-might-not-need-an-effect)
  The single highest-leverage page for interview prep on `useEffect`. Use for: spotting when an effect is wrong.
- [React Docs — "Synchronizing with Effects"](https://react.dev/learn/synchronizing-with-effects)
  Re-frames effects as synchronization, not lifecycle. Use for: the core mental model.
- [Dan Abramov — "A Complete Guide to useEffect" (overreacted.io)](https://overreacted.io/a-complete-guide-to-useeffect/)
  The canonical deep dive. Long but worth it. Use for: closures, dependencies, the "each render has its own everything" model.
- [Dan Abramov — "Before You memo()" (overreacted.io)](https://overreacted.io/before-you-memo/)
  Use for: understanding when `useMemo` / `useCallback` actually help vs when restructuring components is better.
- [Josh Comeau — "Understanding useEffect"](https://www.joshwcomeau.com/react/the-bart-simpson-guide-to-react-hooks/)
  Friendly explanations with strong analogies. Use for: alternative explanations when the official docs don't click.
- [Kent C. Dodds — "Don't Sync State. Derive It!"](https://kentcdodds.com/blog/dont-sync-state-derive-it)
  Use for: anti-pattern detection in interview code samples.

## Wisdom (Communities)

- [r/reactjs](https://reddit.com/r/reactjs)
  High-signal subreddit. Use for: critique of patterns you're unsure about, plus mock interview question threads.
- [Reactiflux Discord](https://www.reactiflux.com/)
  Live chat with experienced React engineers. Use for: real-time questions when stuck.
